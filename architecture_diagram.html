Data Flow Legend
User uploads resource → Backend → Processing Layer
Text → Topic Extraction → Quiz Generation Engine
Quiz Gen → Distractor Module → Filtered Questions
Filtered Questions → Database (SQLite/MySQL)
User Attempts Quiz → Attempt stored in DB → Score computed
Evaluation Engine reads Questions → Generates metrics & graphs
Admin releases results → Celery dispatches email notifications
Fig. 1. Proposed seven-layer architecture of the Automatic Quiz Generation and Evaluation System. The system integrates a Large Language Model (LLM) inference engine with a semantic Distractor Optimization Module (Layer 3d) that employs Sentence-BERT embeddings and TF-IDF cosine similarity to generate plausible, diverse, and non-redundant distractors. A dedicated standalone Evaluation Metrics Engine (Layer 4) quantifies distractor plausibility, lexical diversity, context relevance, and duplicate question rate, producing research-grade CSV reports and correlation heatmaps. Asynchronous task execution via Celery ensures non-blocking quiz generation and timely result notifications.
